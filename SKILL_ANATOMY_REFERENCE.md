# Skill Anatomy Reference

This reference explains the standard v0.6 skill format.

## Purpose
The promise of the skill and the kind of result it owns.

## Use when
Positive activation signals. These should help routing.

## Avoid when
Misuse cases. This section prevents a skill from becoming a generic catch-all.

## Inputs to gather
The minimum context that changes the answer.

## Operating rules
Hard behavioral constraints inside the skill.

## OpenClaw tool pattern
How the skill should interact with workspace context, external evidence, or execution surfaces.

## Expanded workflow
The default ordered path the skill should follow.

## Output contract
What the result must contain to be usable.

## Failure modes to avoid
What weak or dangerous usage looks like.

## Handoff cues
What another operator needs if the task continues later.

## Example invocation
Slash-style and natural-language activation cues.

## Quality bar
What separates an acceptable pass from a strong pass.
